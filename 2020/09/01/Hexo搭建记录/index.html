博客配置小记（零）：博客的创建和初始化

本文最后更新于：3 天前

前言

Jekyll的布置失败以后，考虑到视频资源等的缘故，考虑使用Hexo进行博客的搭建。

Hexo博客的初步搭建主要参考了Codesheep羊哥的视频，非常感谢羊哥的帮助！

初始化

什么算初始化呢？在我看来只要能本地预览，上传到GitHub还能看，就算是初始化成功了。至于说图片、公式，或者说利用Typora写博客，这些都是后话了。

注：若以下未做标记，则都是在主目录下进行。

前期准备

  • Nodejs，最好选择稳定版
  • git 仓库并配置，配置好SSH和全局变量（之前配置Jekyll已经配好了）
  • npm或cnpm
  • vim（可选）

cnpm

一般情况是使用 npm 进行 hexo 的安装的，但是由于国内 npm 速度比较慢，所以考虑使用 cnpm 进行安装。

输入 npm install -g cnpm --registry=https://registry.npm.taobao.org，用 cnpm 代替默认的 npm 。

常用命令

安装Hexo

使用 cnpm install -g hexo-cli 进行安装，安装之后可以用 hexo -v 进行验证。

博客初步搭建

建立博客文件夹

输入 pwd ，可以查看当前路径（在 Ubuntu 中直接就显示了当前的路径，而 Mac 中不作显示，所以视频里需要输入 pwd 进行查看）

mkdir myblog ，在主目录下生成 myblog。

cd myblog ，进入目录，以下都在这个目录。

初始化

sudo hexo init 进行博客的初始化。

hexo s ，输入之后在 localhost：4000 显示页面

第一篇文章

hexo n "test" ，即生成了第一篇文章。

可以发现在 myblog/source/_posts 下生成了 test.md 。

注意，博文生成时都会有如下的头格式，没有这个格式，网页中显示会不完全。所以不能直接 拷贝 .md 过来，需要加上这个格式。

编辑博文时，用 gedit 感觉不是很方便，目前还在探索中。

生成页面

hexo clean 清除缓存文件

hexo g 进行页面的生成

hexo s 本地启动

部署到 GitHub

1.创建一个仓库 hexotest.github.io 。

2.安装cnpm install --save hexo-deployer-git ，一定要在 myblog目录下！不然就会报错说找到不到 git 。

3.设置 _config.yml ，更改 Deployment ，尤其需要主义格式（空格问题）！！

  1. hexo d 部署，输入帐号密码就行了。

加载本地图片

个人比较执着于使用本地图片而不是图库，因为设想是利用 Typora 进行写作，所以也探索了如何加载本地图片。

因为打算以本地图片为主，所以设置了_config.yml 中 Writing -post_asset_folder 的值为 true ，这样生成新博文的时候会在 _posts 下生成与博客同名的文件夹，可以方便图片的存放和整理。

我们知道 markdown 插入图片的语句是 ![描述，可选](路径)

对于网页上的图片 ，路径 可以右键单击 ，点击 复制图片地址 ，即可得到。

对于本地图片却是颇为费神。

先是安装了 hexo-asset-image 这个插件，后来发现原因可能是该插件现在（2020-08-29）的版本是 1.0.0，可能与当前的 hexo 兼容性不好（前端知识比较匮乏，具体原因不大明白）。
